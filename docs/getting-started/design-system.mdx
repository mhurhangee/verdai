# Verda Design System

This document outlines the design principles and components that make up Verda's user interface.

## Layout Structure

### Hub Page Layout

All pages in the hub should use the `HubPage` component which provides:

- Consistent container widths (default: max-w-5xl, fullWidth: max-w-screen-xl)
- Standardized padding and spacing (py-6 px-4 sm:px-6)
- Sidebar toggle button integrated into the page header
- Breadcrumb navigation for wayfinding
- Consistent header with icon, title, description and actions
- Responsive adaptations for mobile and desktop

```tsx
<HubPage
  title="Page Title"
  icon={<Icon size={24} />}
  description="Page description text"
  breadcrumbs={[
    { label: 'Section', href: '/hub/section' },
    { label: 'Current Page', href: '/hub/section/page' }
  ]}
  primaryAction={<PrimaryButton />} // Main action button (e.g., "New Project")
  actions={<OtherActionComponents />} // Additional actions if needed
  fullWidth={false} // Set to true for max-w-screen-xl
>
  {/* Page content */}
</HubPage>
```

### Data Display Components

Verda provides flexible components for displaying data in both list and grid views:

#### DataDisplay

The core component that handles both list and grid views with optional view toggle:

```tsx
<DataDisplay
  items={dataItems}
  viewMode={viewMode} // 'list' or 'grid'
  onViewModeChange={setViewMode}
  persistKey="uniqueKey" // Saves view preference in localStorage
  emptyMessage="No items found"
  renderItem={(item, currentViewMode) => (
    <DataItem
      key={item.id}
      viewMode={currentViewMode}
      title={item.title}
      description={item.description}
      icon={<ItemIcon />}
      // Additional props...
    />
  )}
  groupBy={item => item.category} // Optional grouping function
  groupLabel={(groupKey, items) => <GroupHeader label={groupKey} />}
/>
```

#### DataItem

A versatile item component that adapts to list or grid view:

```tsx
<DataItem
  viewMode="list" // or "grid"
  title="Item Title"
  description="Item description"
  icon={<ItemIcon />}
  metadata={<MetadataDisplay />}
  tags={<TagsDisplay />}
  badges={<BadgesDisplay />}
  actions={<ActionButtons />}
  footer={<FooterContent />}
  linkTo="/item/details" // Optional navigation link
  onClick={handleClick} // Optional click handler
/>
```

#### Feature-specific Data Views

- `ProjectsDataView`: For displaying projects in list/grid view
- `FilesDataView`: For displaying files in list/grid view
- `DocsDataView`: For displaying documentation in list/grid view

These components encapsulate the data display logic for each feature and provide a consistent interface.

### Content Section Pattern

Use the Card component to organize page content into logical sections:

```tsx
<Card className="mb-8 w-full">
  <CardHeader className="flex flex-row items-center justify-between">
    <CardTitle>Section Title</CardTitle>
    <Button>Action</Button>
  </CardHeader>
  <CardContent>
    {/* Section content */}
  </CardContent>
</Card>
```

## Typography

- Page titles: text-3xl font-bold tracking-tight
- Section titles: text-xl font-semibold
- Descriptions: text-muted-foreground text-lg
- Meta text: text-muted-foreground text-sm/text-xs

## Responsive Design

- Mobile breakpoint: 768px
- Desktop adaptations:
  - Change flex-col to flex-row
  - Adjust spacing and padding
  - Grid layouts: grid-cols-1 md:grid-cols-2 lg:grid-cols-3/4
  
## Action Buttons

Place primary action buttons in the `primaryAction` prop of HubPage:

```tsx
// Page component
<HubPage
  primaryAction={<FeaturePageAction onAction={handleAction} />}
  /* other props */
>
  <FeaturePageContent />
</HubPage>

// Feature action component (in FeaturePageContent.tsx)
export function FeaturePageAction({ onAction }: { onAction?: () => void }) {
  return <ActionComponent onAction={onAction} />
}
```

Mobile adaptations:
- Action buttons appear at the top-right on mobile for easy access
- Actions appear next to the title on desktop layouts

## Data View Toggle

Use a switch component to toggle between list and grid views:

```tsx
const [viewMode, setViewMode] = useState<ViewMode>('list')

// In your component's return statement
<div className="flex items-center gap-2">
  <Label htmlFor="view-mode" className="text-sm">List</Label>
  <Switch 
    id="view-mode" 
    checked={viewMode === 'grid'}
    onCheckedChange={(checked) => setViewMode(checked ? 'grid' : 'list')}
  />
  <Label htmlFor="view-mode" className="text-sm">Grid</Label>
</div>

<DataDisplay
  items={items}
  viewMode={viewMode}
  onViewModeChange={setViewMode}
  persistKey="uniqueKey" // Automatically persists user preference
  // ...other props
/>
```

Features:
- View preference is saved to localStorage
- User-friendly switch toggle in the page header
- Items adapt their appearance based on the current view mode
- Consistent styling across both view modes
- Responsive grid layout for different screen sizes

## Metadata Display

Use the `MetadataDisplay` component for consistent metadata presentation:

```tsx
import { MetadataDisplay } from '@/components/shared'
import { formatDateStandard } from '@/lib/utils'
import { ClockIcon, UserIcon, FileIcon } from 'lucide-react'

// Create metadata items with icons
const metadataItems = [
  {
    icon: <ClockIcon className="h-3.5 w-3.5" />,
    content: `${formatDateStandard(item.updatedAt)}`
  },
  {
    icon: <UserIcon className="h-3.5 w-3.5" />,
    content: item.author
  },
  {
    icon: <FileIcon className="h-3.5 w-3.5" />,
    content: `${fileCount} files`
  }
]

// Display the metadata
<MetadataDisplay items={metadataItems} />
```

### Metadata Standards

- **Dates:** Use ClockIcon and format as "dd MMM yyyy"
- **File size:** Use HardDriveIcon and format in MB
- **File count:** Use FileIcon with consistent pluralization
- **Authors:** Use UserIcon 
- **Categories:** Use FolderIcon
- **Reading time:** Use TimerIcon

For detailed guidelines, see [Metadata Display](/hub/docs/design-system/metadata).

## Icons

- Standard icon size: 24px for page icons, 16px for UI elements
- Metadata icons: 14px (h-3.5 w-3.5)
- Use Lucide icons throughout the application
- Primary action icons should use text-primary color

## Breadcrumbs

Use breadcrumbs for wayfinding and navigation:

```tsx
breadcrumbs={[
  { label: 'Parent', href: '/hub/parent' },
  { label: 'Child', href: '/hub/parent/child' }
]}
```

## Component Spacing

- Between sections: mb-8
- Between elements: gap-4
- Internal padding: p-4 sm:p-6
